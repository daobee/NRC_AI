### 3.2 核对工具

编写脚本自动化对比：

```python
def compare_pokemon_stats(pokemon_name, nrc_stats, source_stats):
    """比对六维数值"""
    mismatches = {}
    for stat in ["hp", "atk", "def", "spatk", "spdef", "speed"]:
        if nrc_stats[stat] != source_stats[stat]:
            mismatches[stat] = (nrc_stats[stat], source_stats[stat])
    return mismatches
```

**执行**：
```bash
python scripts/calibrate_pokemon_stats.py --sample 50
```

**输出**：

- 偏差分布直方图
- 需要修正的精灵列表

## 4. 偏差分析

### 4.1 接受标准

| 偏差大小 | 处理方案 |
|--------|--------|
| ±0 | ✅ 完全匹配 |
| ±1-2 | ⚠️ 可能是四舍五入，接受 |
| ±3-5 | ❌ 需调查原因 |
| ±6+ | ❌ 数据有误，必须修正 |

### 4.2 常见偏差原因

1. **版本差异** — 游戏更新时数值改动
2. **四舍五入** — 数值存储精度差异
3. **双属性加成** — 某些属性有隐藏加成
4. **特性修正** — 特定特性在加载时修改值
5. **数据导入错误** — 原始 Excel/JSON 输入错误

---

## 5. 修正流程

若发现偏差 ≥ ±3：

### 5.1 根因调查

1. 查看 `pokemon_db.py` 加载逻辑
2. 检查 `nrc.db` 源表 (原始 SQL 导入)
3. 确认游戏内版本与 DB 版本是否同步
4. 查看特性或其他自动修正是否被应用

### 5.2 修正方案

**选项 A**: 更新 nrc.db 中的值

```sql
UPDATE pokemons SET hp = ??? WHERE name = '精灵名'
```

**选项 B**: 在代码中添加修正规则

```python
# src/pokemon_db.py
STAT_CORRECTIONS = {
    "精灵名": {"hp": +5, "atk": -2},
}
```

**建议**: 选择 A（更正数据库源）

### 5.3 修正后验证

- 重新运行校准脚本
- 确认所有偏差 ≤ ±1
- 提交修改到 git

## 6. 伤害计算的影响

### 6.1 敏感度分析

精灵六维数值对伤害的影响权重：

| 字段 | 影响公式项 | 影响权重 | 关键性 |
|-----|----------|--------|------|
| **攻击** | (atk/def) | 100% | 🔴 |
| **防御** | (atk/def) | 100% | 🔴 |
| **特攻** | (spatk/spdef) | 100% | 🔴 |
| **特防** | (spatk/spdef) | 100% | 🔴 |
| **速度** | 行动顺序 | 50%* | 🟠 |
| **HP** | 回复百分比 | 10%** | 🟡 |

*速度仅影响先手判定，不影响伤害数值  
**仅影响 HEAL_HP 等回复效果

### 6.2 校准优先级

1. **最高**: 攻/防/特攻/特防 — 直接影响伤害
2. **次高**: 速度 — 影响战斗流程
3. **最低**: HP — 仅影响生存判定和回复量
