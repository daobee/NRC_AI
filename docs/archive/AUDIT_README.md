### 快速查询已配置特性

查看已配置的 31 个特性:
```bash
grep -E "\"[^\"]+\": \[" src/effect_data.py | grep -A 5 "ABILITY_EFFECTS"
```

### 快速查询未配置特性

列出所有 138 个未配置特性:
```python
python3 -c "
import sqlite3
from src.effect_data import ABILITY_EFFECTS

conn = sqlite3.connect('data/nrc.db')
cursor = conn.cursor()
cursor.execute(\"SELECT DISTINCT ability FROM pokemon WHERE ability != ''\")
db_abilities = [row[0].split(':')[0] for row in cursor.fetchall()]
conn.close()

configured = set(ABILITY_EFFECTS.keys())
unconfigured = sorted(set(db_abilities) - configured)
for ab in unconfigured:
    print(ab)
"
```

## 📊 关键洞察

### 为什么特性覆盖率这么低?

特性配置比技能更复杂:
- **技能**: 通常是单一效果 (造成伤害、回复等)
- **特性**: 常涉及复杂逻辑 (条件判断、计时、计数等)

因此特性需要手工实现，无法完全自动生成。

### 自动生成的好处

自动生成脚本 (scripts/generate_skill_effects.py) 成功覆盖了 91.9% 的技能：
- 减少手工配置工作
- 保持一致性
- 快速迭代

### 为什么有 8 个重复技能?

这是设计好的：
- **手工配置**: 优先级高，逻辑更复杂
- **自动生成**: 提供备选方案

需要验证加载顺序，确保使用手工配置。
