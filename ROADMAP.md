## 优先级排序

### 🥇 P1 — 技能 / 特性效果配置（最高优先）

这是阶段一的**核心工作**，直接决定模拟器还原度。

#### 1A. 特性补全（102 个待配）

已完成 68/170（40%），架构已就位，每个新特性 = 配置 + handler + 测试。

```
开发流程：
1. effect_models.py  → E 枚举新增（如果需要新原语）
2. effect_data.py    → ABILITY_EFFECTS["特性名"] = [AE(Timing.xxx, [T(E.xxx, ...)])]
3. effect_engine.py  → def _h_xxx(tag, ctx): ... handler
4. tests/            → 测试
```

#### 1B. 技能效果修正（42 个空值 + 质量审计）

- 42 个自动生成技能效果为空 `[]`，需逐个判断：确实无效果还是遗漏
- 部分自动生成的效果可能与游戏描述不完全一致，需抽查校正

#### 1C. 伤害公式校准（需要游戏内数据）

- 当前公式：`(攻/防) × 0.9 × 威力 × 能力等级 × 威力提升 × 本系 × 克制 × 天气 × (1-减伤)`
- 攻防比例系数 0.9 是否准确？需要 20+ 组游戏内真实伤害对照数据
- 精灵六维数值需与游戏内核对

## 项目结构

```
NRC_AI/
├── src/                        # 核心源码（12 个文件）
│   ├── models.py               # Pokemon / Skill / BattleState 数据模型
│   ├── effect_models.py        # E 枚举（效果原语）/ Timing / SkillTiming
│   ├── effect_data.py          # 手工技能效果 + 特性效果配置（核心数据文件）
│   ├── effect_engine.py        # 效果执行引擎（Handler 注册表）
│   ├── skill_effects_generated.py  # 自动生成技能效果
│   ├── battle.py               # 战斗逻辑 + 印记/回合流程
│   ├── skill_db.py / pokemon_db.py # 数据库加载
│   ├── mcts.py                 # MCTS AI
│   └── server.py               # FastAPI + WebSocket
├── web/                        # 前端（3 个 HTML 文件）
│   ├── battle.html             # 战斗界面（40KB，含 CSS + JS）
│   └── team.html               # 阵容编辑器
├── tests/                      # 测试（14 个文件，107 个用例）
├── data/nrc.db                 # SQLite 数据库（461 精灵 × 495 技能）
├── scripts/                    # 工具脚本（爬虫/生成器/审计）
├── docs/                       # 参考文档
│   ├── COVERAGE_MATRIX.md      # 特性覆盖矩阵（已配 vs 待配详细清单）
│   ├── SKILLS_ABILITIES_CONFIG_GUIDE.md  # 配置开发手册
│   └── archive/                # 历史文档归档
```

## 核心公式参考

```
伤害 = (攻/防) × 0.9 × (威力 + 威力加成) × 能力等级 × 威力提升buff × 本系1.5 × 克制 × 天气 × (1-减伤)
能力等级 = (1 + 我方攻升 + 敌方防降) / (1 + 我方攻降 + 敌方防升)
```

## 新增特性标准流程

```
1. effect_models.py   → E.XXX = auto()             # 新原语（仅在需要时）
2. effect_data.py     → ABILITY_EFFECTS["特性名"]   # 配置
3. effect_engine.py   → def _h_xxx(tag, ctx): ...  # handler
4. tests/             → 测试验证
5. battle.py          → 不动（只在正确时机调用 execute_ability）
```
